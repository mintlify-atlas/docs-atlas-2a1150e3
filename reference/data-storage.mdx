---
title: "How TransactApp stores your financial data"
description: "TransactApp uses a local SQLite database to store your transactions and cash inventory. Your data stays on your device and is never transmitted to any server."
---

TransactApp stores all of your financial data locally using SQLite, a lightweight embedded database engine. There is no server, no cloud account, and no data transmitted off your machine. Everything — your transactions, cash inventory records, and app settings — lives in a single database file on your Windows PC.

## Storage overview

| Component | Detail |
|---|---|
| **Database engine** | SQLite (via System.Data.SQLite 2.0.3 / SourceGear.sqlite3 3.50.4.2) |
| **Data location** | User's local application data folder (ClickOnce isolated storage) |
| **File format** | SQLite database file (`.db` or `.sqlite`) |
| **JSON serialization** | Newtonsoft.Json 13.0.4 (used internally for data handling) |
| **Server required** | None — fully local |

## Frequently asked questions

<AccordionGroup>
  <Accordion title="Where is my data stored?">
    ClickOnce applications store their data in Windows isolated storage, typically under a path such as:

    ```
    %LOCALAPPDATA%\Apps\2.0\...
    ```

    The exact path includes a hashed folder name generated by Windows based on the application's identity. You can locate your data directory by navigating to `%LOCALAPPDATA%\Apps\2.0\` in File Explorer and looking for a folder associated with TransactApp.

    Because the path is managed by ClickOnce, it changes if you reinstall the application. Always export or back up your database file before uninstalling.
  </Accordion>

  <Accordion title="What data does TransactApp store?">
    The SQLite database contains:

    - **Transactions** — income and expense records you have entered
    - **Cash inventory** — physical cash denomination counts and totals
    - **App settings** — preferences and configuration saved between sessions

    No data is collected, transmitted, or stored anywhere outside your local machine.
  </Accordion>

  <Accordion title="Is my data safe?">
    Your data never leaves your device. TransactApp does not connect to any external server to read or write your financial records. The SQLite database file is stored under your Windows user profile, which means other users on the same PC cannot access it without your credentials.

    That said, the database file itself is not encrypted. If your Windows user account is compromised or your device is accessed physically, the data file could be read directly. For sensitive financial data, consider enabling Windows BitLocker on your drive for full-disk encryption.
  </Accordion>

  <Accordion title="Can I move my data to another computer?">
    Yes. To transfer your data:

    1. Locate the SQLite database file in `%LOCALAPPDATA%\Apps\2.0\`.
    2. Copy the file to external storage or a transfer medium.
    3. Install TransactApp on the new machine.
    4. Replace the new machine's database file with your copy.

    Make sure TransactApp is closed before copying the database file to avoid data corruption.
  </Accordion>

  <Accordion title="What happens to my data if I uninstall TransactApp?">
    Uninstalling a ClickOnce application through Windows may or may not remove the application data folder, depending on how the uninstall is performed. To be safe, back up your database file **before** uninstalling. See the backup guide linked below for step-by-step instructions.
  </Accordion>
</AccordionGroup>

## Technical details

TransactApp uses the following SQLite stack:

- **System.Data.SQLite 2.0.3** (via `Stub.System.Data.SQLite.Core.NetStandard 1.0.119`) — the primary ADO.NET provider
- **SQLitePCLRaw 3.0.2** (core, bundle, and provider packages) — low-level SQLite bindings
- **SourceGear.sqlite3 3.50.4.2** — native SQLite library (version 3.50.4)
- **SQLite 3.13.0** — base SQLite version reference

Native binaries are bundled for `win-x64`, `win-x86`, and `win-arm64`, so no separate SQLite installation is required.

<Note>
  Regularly backing up your database file protects you from data loss due to hardware failure, accidental deletion, or issues during an app update. See the [data backup guide](/guides/data-backup) for instructions on creating and restoring backups.
</Note>
